Privacy Policy

Last updated: February 27, 2026

Kuurier is designed to protect your privacy. This policy explains what data we handle, what we don't, and how the system is built to minimize data collection by design.

Our Core Principle

We cannot access your data because we never have it. Kuurier is built so that the server stores only what is necessary for message routing and never stores personally identifiable information.

What We Do NOT Collect

  • No email addresses — authentication uses Ed25519 cryptographic signatures
  • No phone numbers — your keypair is your identity
  • No names or usernames — you are identified by your public key
  • No passwords — challenge-response authentication only
  • No IP addresses — the desktop app routes through Tor by default
  • No message content — messages are end-to-end encrypted with the Signal Protocol; the server stores only ciphertext
  • No analytics or tracking — no third-party SDKs, no telemetry, no usage tracking
  • No advertising — Kuurier has no ads and no advertising partnerships

What the Server Stores

The Kuurier server stores the minimum data required to operate:

  • Your Ed25519 public key (32 bytes) — used to verify your identity during authentication
  • Signal Protocol key bundles — public keys used for establishing encrypted sessions with other users
  • Encrypted message ciphertext — opaque blobs that the server cannot decrypt; stored temporarily until delivered
  • Trust score — a numeric value based on vouches from other members
  • Invite relationships — which invite code was used during registration (for web-of-trust accountability)
  • Event and alert metadata — event titles, times, and locations you choose to share; alert descriptions and coordinates

Local Device Storage

Your device stores:

  • Ed25519 private key — stored in the OS keychain (macOS Keychain, Windows Credential Manager, or Linux libsecret). Never transmitted.
  • Signal Protocol session state — encryption keys for active conversations
  • Decrypted messages — cached locally in an encrypted SQLite database. The encryption key is derived from your private key.
  • Tor consensus data — cached network information for faster Tor connections

All local data can be destroyed instantly using the Panic Wipe feature.

Tor and Network Privacy

The desktop app includes an embedded Tor client. When enabled, all traffic between your device and the Kuurier server is routed through the Tor network. This means:

  • The Kuurier server sees traffic from Tor exit nodes, not your real IP
  • Your ISP sees Tor traffic, but cannot determine you are using Kuurier
  • No network metadata (IP, geolocation) is stored on the server

Third Parties

Kuurier does not share data with any third parties. We do not use analytics services, advertising networks, crash reporting tools, or any external SDKs that phone home.

The only external network connection the app makes is to the Kuurier API server (through Tor, if enabled) and to the Tor network itself.

Data Retention

  • Encrypted messages are stored until delivered to all recipient devices, then marked for cleanup
  • Signal Protocol one-time pre-keys are consumed on use and deleted
  • Device link requests expire after 5 minutes
  • Auth challenges expire after 5 minutes
  • Invite codes expire after 7 days

Account Deletion

You can delete your account at any time via the API (DELETE /me). This cascades to all associated data: devices, keys, vouches, posts, and channel memberships.

Law Enforcement

We cannot provide message content in response to legal requests because we do not have it. Messages are end-to-end encrypted; the server only stores ciphertext that we cannot decrypt. We cannot provide IP addresses because we do not log them.

Open Source

Kuurier is open source. You can audit the server code, client code, and encryption implementation to verify these privacy claims. View source on GitHub.

Changes

If we make changes to this policy, we will update the "Last updated" date at the top. Since Kuurier has no email addresses or contact information for users, policy changes will be announced via the project's GitHub repository.

Contact

For questions about this privacy policy, open an issue on GitHub.

Kuurier is open source software. No personal data is collected.
